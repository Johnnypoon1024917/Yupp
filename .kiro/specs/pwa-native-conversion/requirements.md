# Requirements Document

## Introduction

Convert the YUPP Next.js travel application into an installable, offline-capable Progressive Web App (PWA). The conversion enables a frictionless "Add to Home Screen" experience on iOS and Android, resulting in a native app icon and standalone app shell with full-screen display, no browser chrome, and overscroll prevention.

## Glossary

- **PWA_Engine**: The `@ducanh2912/next-pwa` package — the actively maintained community fork of next-pwa compatible with Next.js App Router — responsible for generating and registering the Service Worker during build.
- **Service_Worker**: A background script generated by the PWA_Engine that intercepts network requests and enables offline caching.
- **App_Manifest**: The `public/manifest.json` file that declares the application name, icons, display mode, colors, and start URL to the browser's PWA install heuristics.
- **Layout_Metadata**: The Next.js `Metadata` and `Viewport` exports in `src/app/layout.tsx` that inject `<meta>` and `<link>` tags into the HTML `<head>`.
- **Next_Config**: The `next.config.mjs` file that configures the Next.js build pipeline, including the PWA_Engine wrapper.
- **Standalone_Mode**: The `display: "standalone"` setting in the App_Manifest that removes browser chrome (address bar, navigation buttons) when the app is launched from the home screen.
- **Install_Prompt**: The browser-native prompt (or iOS "Add to Home Screen" action) that allows users to install the PWA onto their device.

## Requirements

### Requirement 1: Install and Configure PWA Engine

**User Story:** As a developer, I want the PWA build pipeline integrated into the Next.js config, so that a Service Worker is automatically generated on every production build.

#### Acceptance Criteria

1. THE Next_Config SHALL wrap the existing configuration object with the PWA_Engine initializer from `@ducanh2912/next-pwa`.
2. WHEN the PWA_Engine is initialized, THE Next_Config SHALL set the `dest` option to `"public"` so that generated Service Worker files are placed in the public directory.
3. WHEN the PWA_Engine is initialized, THE Next_Config SHALL set `register` to `true` so that the Service Worker is auto-registered in the browser.
4. WHEN the PWA_Engine is initialized, THE Next_Config SHALL set `skipWaiting` to `true` so that updated Service Workers activate immediately.
5. WHILE the application is running in development mode, THE PWA_Engine SHALL be disabled by setting `disable` to `process.env.NODE_ENV === "development"`.
6. THE Next_Config SHALL preserve the existing `serverExternalPackages: ['puppeteer-core']` setting after wrapping with the PWA_Engine.
7. WHEN a production build is executed, THE PWA_Engine SHALL generate a `sw.js` file in the `public` directory.

### Requirement 2: Create Web App Manifest

**User Story:** As a mobile user, I want the app to declare itself as an installable PWA via a manifest file, so that my device recognizes it and offers the install prompt.

#### Acceptance Criteria

1. THE App_Manifest SHALL be located at `public/manifest.json`.
2. THE App_Manifest SHALL set `name` to `"YUPP Travel"` and `short_name` to `"YUPP"`.
3. THE App_Manifest SHALL set `description` to `"The AI-Powered Travel Command Center."`.
4. THE App_Manifest SHALL set `start_url` to `"/"`.
5. THE App_Manifest SHALL set `display` to `"standalone"` to enable Standalone_Mode.
6. THE App_Manifest SHALL set `background_color` to `"#FAFAFA"` and `theme_color` to `"#FAFAFA"` to match the existing application background.
7. THE App_Manifest SHALL set `orientation` to `"portrait"`.
8. THE App_Manifest SHALL include an icon entry with `src`, `sizes` set to `"192x192"`, and `type` set to `"image/png"`.
9. THE App_Manifest SHALL include an icon entry with `src`, `sizes` set to `"512x512"`, and `type` set to `"image/png"`.
10. WHEN parsed as JSON, THE App_Manifest SHALL be valid JSON conforming to the W3C Web App Manifest specification.

### Requirement 3: Inject Native Meta Tags and Manifest Link

**User Story:** As a mobile user, I want the HTML head to contain the correct PWA meta tags, so that iOS and Android treat the app as a native-like standalone application.

#### Acceptance Criteria

1. THE Layout_Metadata SHALL export a `Viewport` object with `width` set to `"device-width"`, `initialScale` set to `1`, `maximumScale` set to `1`, `userScalable` set to `false`, and `themeColor` set to `"#FAFAFA"`.
2. THE Layout_Metadata SHALL set the `title` field to `"YUPP | Travel Planner"`.
3. THE Layout_Metadata SHALL set the `manifest` field to `"/manifest.json"` to link the App_Manifest.
4. THE Layout_Metadata SHALL set `appleWebApp` with `capable` set to `true`, `statusBarStyle` set to `"default"`, and `title` set to `"YUPP Travel"`.
5. THE Layout_Metadata SHALL set the `description` field to `"The AI-Powered Travel Command Center."`.

### Requirement 4: Prevent Overscroll Bounce in Standalone Mode

**User Story:** As a mobile user using the app in standalone mode, I want the app to prevent rubber-band overscroll, so that the experience feels native rather than web-based.

#### Acceptance Criteria

1. THE `body` element in `globals.css` SHALL include `overscroll-behavior: none`.
2. THE `body` element in `globals.css` SHALL include `touch-action: none`.

### Requirement 5: Generate Placeholder PWA Icons

**User Story:** As a mobile user, I want the app to have proper home screen icons, so that the installed PWA has a recognizable icon on my device.

#### Acceptance Criteria

1. THE application SHALL include a PNG icon file at a path referenced by the 192x192 icon entry in the App_Manifest.
2. THE application SHALL include a PNG icon file at a path referenced by the 512x512 icon entry in the App_Manifest.
3. WHEN the icon files are served, THE application SHALL serve valid PNG image data.

### Requirement 6: Gitignore Service Worker Artifacts

**User Story:** As a developer, I want generated Service Worker files excluded from version control, so that build artifacts do not pollute the repository.

#### Acceptance Criteria

1. THE `.gitignore` file SHALL include entries to ignore the generated Service Worker files (`sw.js`, `sw.js.map`, `workbox-*.js`, `workbox-*.js.map`) in the `public` directory.

### Requirement 7: Manifest JSON Round-Trip Integrity

**User Story:** As a developer, I want confidence that the manifest file is always valid JSON, so that browsers can reliably parse it for PWA installation.

#### Acceptance Criteria

1. FOR ALL valid App_Manifest content, parsing the JSON string and re-serializing the resulting object SHALL produce a semantically equivalent JSON string (round-trip property).
